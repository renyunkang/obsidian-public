    # ip 分配

    创建工作负载时指定分配方式

    • 随机分配 对于工作负载的 ip 分配策略

      • 优先使用负载指定的：ovn.kubernetes.io/logical_switch: subnet
      • 如果负载没有指定使用，使用 namespace 绑定的 subnet
      • 如果 namespace 没有绑定 subnet，使用安装默认的 ovn-default

    • 指定 subnet 分配

    1
2

          annotations:
        ovn.kubernetes.io/logical_switch: subnet
    • 静态分配
    1. 所使用的 IP/Mac 不能和已有的 IP/Mac 冲突。
    2. IP 必须在所属子网的 CIDR 内
    3. IP 数量小于 replicas 数量时，多出的 Pod 将无法创建
    1. 为 pod 静态分配 支持为一个 pod 静态分配 ip，指定 subnet 分配之后，设置对应的静态 ip、mac；如果 mac 地址为空，会随机申请出一个 mac 地址，因此设计时可以只允许指定 ip，指定的 ip 必须在 subnet cidr 范围内。
    1
2
3
4

      annotations:
    ovn.kubernetes.io/ip_address: 10.233.128.2
    ovn.kubernetes.io/logical_switch: subnet
    ovn.kubernetes.io/mac_address: 00:00:00:53:6B:B6
    1. 为 workload 静态分配 支持给 Workload 设置固定 IP，包括Deployment/StatefulSet/DaemonSet/Job/CronJob 预计下个版本支持双栈，使用;分割不同pod，使用,分割双栈地址
    1
2
3

          annotations:
        ovn.kubernetes.io/logical_switch: subnet
        ovn.kubernetes.io/ip_pool: 10.233.128.200,10.233.128.201,10.233.128.202

    # ip 预留

    在 ippool 管理页面

    预留的 ip 必须在指定的 subnet cidr 范围内

    kube-ovn 支持两种方式来避免分配某些 ip

    1. 创建 subnet 时可以在 spec.excludeIps 中预留 ip 在子网中指定不能分配的地址或者地址范围，可以为多个，取值如下：
    • 10.233.128.1..10.233.128.10
    • 10.233.128.200
    1. 使用 vip 资源 - 单个的地址预留,在后续的版本中预留的 ip 地址可以指定分配给其他负载 可以指定 ip 预留，不指定时随机预留一个 ip。

    • ips 资源可以查看具体的 ip 分配信息

    # networkPolicy

    现有的网络隔离存在：项目级别的网络隔离以及企业空间的网络隔离。 对于 kube-ovn 可以做到除了NetworkPolicy 之外，还支持子网级别的网络隔离以及更细粒度的访问控制，以及支持安全组。

    1. 【子网隔离】开启子网间的隔离，并自定义添加允许访问的其他子网
    2. 【访问控制】子网支持设置更细粒度的访问控制
      字段说明取值
      action动作allow/reject/drop/allow-related/allow-stateless
      direction方向from-lport/to-lport
      match匹配规则ip4.src==172.30.2.201 && ip4.dst==172.30.3.201
      priority优先级0~32767

    【注】：match 需要符合流表规则： ovn-sb(5) - Linux manual page (man7.org)

    1. 安全组 SecurityGroup 更多的应用于将 pod 暴露到外界时的安全访问，是否需要集成
      字段说明取值
      ingressRules/egressRules指定出口/入口规则
      allowSameGroupTraffic是否允许与安全组通信true/false
      ipVersionip 协议ipv4/ipv6
      policy策略allow/drop
      priority优先级，值越小优先级越高0~200
      protocol协议tcp/udp/icmp/all
      portRangeMin端口范围0~65535
      portRangeMax端口范围，可以等于portRangeMin 值0~65535
      remoteType作用的对象类型address/securityGroup
      remoteSecurityGroup当remoteType取securityGroup，指定其名称string

    当 protocol 指定 tcp/udp 时，需要指定 portRange

    通过设置 pod 注解来使用安全组：

    1
2
3
4

    metadata:
  annotations:
    ovn.kubernetes.io/port_security: 'true'
    ovn.kubernetes.io/security_groups: 'sg1,sg3,sgx'

    # qos 带宽限速

    创建负载时设置 1.9 版本之前使用注解控制 pod 的双向带宽：

    1
2
3
4

    metadata:
  annotations:
    ovn.kubernetes.io/ingress_rate: "3" 
    ovn.kubernetes.io/egress_rate: "1"

    1.9 版本之后支持了两种 qos 配置，但是对于带宽的限制仍需要以上注解

    # 双栈支持

    首先底层 k8s 需要修改以支持双栈： IPv4/IPv6 dual-stack | Kubernetes 在配置双栈时，只需要设置对应子网 CIDR 格式为 cidr=<IPv4 CIDR>,<IPv6 CIDR> 即可。 CIDR 顺序要求 IPv4 在前，IPv6 在后。 目前K8S官方在 v1.20版本之后支持Service的双栈，Kube-OVN还没有对Service进行双栈适配。
